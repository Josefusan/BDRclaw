Cold Email
Gmail OAuth

Multi-step sequences, reply detection, meeting link injection. Up to 3 sending accounts with per-account daily limits.

Works today — run npm run gmail-auth
LinkedIn DMs
Browser automation

Connection requests with personalized notes, DM sequences for 1st-degree connections, reply polling. Daily limits enforced.

Run npm run linkedin-auth
Twitter / X DMs
Twitter API v2

DMs via Twitter API v2. Resolves @handles to user IDs automatically. Requires Basic dev tier ($100/mo).

Run npm run twitter-auth
WhatsApp
Twilio WhatsApp API

Outbound messages + inbound reply webhook. Use Twilio sandbox for dev, dedicated number for production. Shared credentials with SMS.

Set TWILIO_WHATSAPP_NUMBER
Telegram
Telegram Bot API

Long-polling bot — works behind any firewall, no webhook needed. Prospects DM the bot; BDRclaw responds and sequences them.

Set TELEGRAM_BOT_TOKEN
SMS
Twilio Programmable SMS

Send and receive SMS. TCPA-compliant — max 2 unsolicited touches per prospect. Reply webhook at /webhooks/sms.

Set SMS_ENABLED=true
Instagram
Instagram Graph API

Warm-lead follow-up only — Meta's API permits DMs only to users who message you first. Perfect for inbound lead nurturing.

Set INSTAGRAM_ENABLED=true

Up in 4 steps

Start with cold email in under 10 minutes. Add channels as you go.

1
Clone + install
git clone github.com/Josefusan/BDRclaw
cd BDRclaw && npm install
2
Configure .env
cp .env.example .env
# Fill in ANTHROPIC_API_KEY + channels
3
Import your leads
npm run import-csv prospects.csv
# or add via dashboard at :3000
4
Start + fire first cycle
npm run dev     # dashboard at :3000
npm run brain   # trigger first outreach

Small enough to understand

One Node.js process. A handful of source files. Agents run in isolated containers — Bash access is safe because commands run inside Docker, not on your host.

# Architecture
Channels (Email / LinkedIn / WhatsApp / SMS / Telegram / Twitter)
↓
SQLite queue
↓
Polling loop + BDR Brain
↓
Container (Claude Agent SDK) ←→ prospects/*/CLAUDE.md
↓
CRM sync + outbound actions

Deploy anywhere

Railway, Fly.io, Docker, bare VPS — pick your flavor. One railway.json and a Dockerfile are included.

Railway
Push the repo. Set env vars in the Railway panel. Done.
Fly.io
fly launch detects the Dockerfile. Mount a volume for /app/store.
Docker
docker build + run. Mount ./store for persistence.
Set Twilio webhook URLs to your deployed domain:
WhatsApp:  https://bdrclaw.dev/webhooks/whatsapp
SMS:       https://bdrclaw.dev/webhooks/sms
